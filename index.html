How to do HTML code

What you need

  • A plain-text editor (VS Code, Sublime Text, Notepad++, or even Notepad).
  • A modern browser (Chrome, Firefox, Edge, Safari) to open and test the page.
  • A folder on your computer to keep the files (e.g., my-site/).

1 — Create the HTML file

  1. Open your editor.
  2. Create a new file and save it as index.html inside your folder.
  3. Paste this minimal HTML skeleton:
<!DOCTYPE html>
	<html lang="en">
	<head>
	    <meta charset="UTF-8">
	    <meta name="viewport" content="width=device-width, initial-scale=1.0">
	    <title>My First Page</title>
	</head>
	<body>
	    <h1>Hello, world!</h1>
	</body>
	</html>

Here is a full example:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />
  <title>My First Page</title>
  <link rel="stylesheet" href="styles.css" /> <!-- optional -->
</head>
<body>
  <header>
    <h1>Welcome to my page</h1>
  </header>

  <main>
    <p>Hello — this is my first HTML page.</p>
    <a href="https://example.com" target="_blank" rel="noopener">Visit example.com</a>
    <img src="photo.jpg" alt="Description of photo" />
  </main>

  <footer>
    <small>© 2025 Your Name</small>
  </footer>

  <script src="script.js"></script> <!-- optional -->
</body>
</html>

2 — What each part means (quick)

  • <!DOCTYPE html> — tells the browser this is HTML5.
  • <html lang="en"> — root element; lang helps accessibility and search engines.
  • <head> — meta information (title, encoding, CSS links).
  • <meta charset="utf-8"> — sets text encoding (use always).
  • <meta name="viewport"...> — makes the page responsive on mobile.
  • <title> — text shown in the browser tab.
  • <link rel="stylesheet" href="styles.css"> — links an external CSS file.
  • <body> — visible page content (headings, paragraphs, images...).
  • <header>, <main>, <footer> — semantic elements that describe page structure.
  • <script src="script.js"> — loads JavaScript (optional).

3 — Add basic content (examples)

Headings and text:

<h1>Main heading</h1>
<h2>Subheading</h2>
<p>This is a paragraph of text.</p>

Lists:

<ul>
  <li>Item A</li>
  <li>Item B</li>
</ul>

<ol>
  <li>First</li>
  <li>Second</li>
</ol>

Image:

<img src="images/photo.jpg" alt="a clear description" width="600" />

Link:

<a href="about.html">About page (internal)</a>
<a href="https://example.com" target="_blank" rel="noopener">External site</a>

Form (basic):

<form action="/submit" method="post">
  <label for="name">Name</label>
  <input id="name" name="name" type="text" required />
  <button type="submit">Send</button>
</form>

4 — Add styling with CSS

Create styles.css in the same folder and add:

/* styles.css */
* { box-sizing: border-box; }
body { font-family: Arial, sans-serif; margin: 0; padding: 0; line-height: 1.5; }
header { background: #f2f2f2; padding: 1rem; }
main { padding: 1rem; }
img { max-width: 100%; height: auto; display: block; }

The <link rel="stylesheet" href="styles.css"> in your HTML will load this file.

5 — Add interactivity with JavaScript

Create script.js:

// script.js
document.addEventListener('DOMContentLoaded', function () {
  const h1 = document.querySelector('h1');
  if (h1) h1.addEventListener('click', () => alert('You clicked the heading!'));
});

This runs after the page loads and shows a simple click behavior.

6 — View and test locally

  • Double-click index.html or drag it into your browser to open it.
  • Use Developer Tools (F12) → Console / Elements to debug layout, inspect elements, and see JS errors.
  • For features that require a server (fetch, APIs), run a local server: e.g., in terminal inside the folder python -m http.server 8000 (then open http://localhost:8000).

7 — Accessibility & best practices

  • Always provide alt text for images.
  • Use semantic tags (<nav>, <main>, <article>, <section>, <aside>) — they help screen readers and SEO.
  • Ensure form inputs have <label>s.
  • Keep heading order logical: <h1> → <h2> → <h3>.
  • Use rel="noopener noreferrer" when opening external links in new tabs (target="_blank").
  • Add lang attribute to <html> and proper meta charset.
  • Make text readable (contrast, font-size) and mobile-friendly (meta viewport + responsive CSS).

8 — Validation, performance and security

  • Validate HTML/CSS with a validator (search “HTML validator” / “WCAG checklist”).
  • Optimize images (resize, compress) to reduce load time.
  • Serve files over HTTPS when publishing.
  • Avoid inline JS for security reasons (CSP considerations) in larger projects.

10 — Deploy (quick options)

  • Upload to a web host or use free hosting: GitHub Pages, Netlify, Vercel (search these names).
  • For GitHub Pages: push the repo and enable Pages in the repo settings (works well for static sites).

Cheatsheet — common HTML tags

<!DOCTYPE html> — start document
<html> — root
<head> — meta + title + links
<body> — visible content
<h1>…<h6> — headings
<p> — paragraph
<a href=""> — link
<img src="" alt=""> — image
<ul>, <ol>, <li> — lists
<form>, <input>, <button>, <label> — forms
<div> — generic container
<header>, <nav>, <main>, <footer>, <section>, <article>, <aside> — semantic layout
